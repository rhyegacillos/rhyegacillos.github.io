AI Automation Engineering

Recent private-client operations automation work focused on the parts of a business that usually live in spreadsheets, admin screens, and repeated follow-ups. The work was about turning those steps into clear tools that operators can trust, understand, and run without needing a developer beside them.

Warehouse Inventory Sync

The warehouse needed one clear path from scanning an order to updating inventory. I built a workflow where staff can scan items, record completion, attach proof, and see the status of each order without jumping between disconnected tools.

The main problem was reliability: missed deductions, duplicate updates, and unclear handoffs. The automation adds source rules, status write-back, audit logs, review dashboards, and operator approval points so the team can see what happened and fix exceptions safely.

Built with AppSheet, Google Sheets, Apps Script, ecommerce APIs, and ERP/inventory APIs.

B2B Account Import Tool

A large business-account import would have taken hours of careful manual entry. I built a spreadsheet-driven tool that lets operators paste account data, validate it, preview the setup, then run the import with clear row-by-row results.

The tool handles ID generation, existing-contact matching, account creation or update, ordering approval, and verification columns. If something fails, the sheet shows exactly which row needs attention instead of stopping the whole batch.

Built with Google Sheets, Apps Script, GraphQL APIs, dry-run mode, rate limiting, retry/backoff, idempotent re-runs, status coloring, and append-only run logs.

Procurement Reorder Dashboard

Purchasing decisions were spread across exports, formulas, stock checks, and open purchase orders. I built a dashboard that brings those inputs together and gives operators a practical answer: what needs to be ordered, how many units, and when action is needed.

The dashboard combines sales history, current stock, open orders, lead times, item mappings, and forecast logic. It also highlights data issues and groups recommendations so purchasing can review the work instead of rebuilding the calculation by hand.

Built with Apps Script, Google Sheets, channel APIs, ERP/inventory APIs, forecast logic, supplier grouping, and dashboard controls for discontinued items.

Shipping Manifest Automation

The shipping team needed a simple daily record of what was handed to the carrier. I built a manifest workflow that gathers shipping records, creates a printable PDF, saves it, and notifies the operator when it is ready.

This gives the business a signed handoff record when tracking scans are late or missing. It also supports held packages, restored packages, and package removal so the printed manifest matches what is actually leaving that day.

Built with Apps Script, shipping APIs, marketplace shipping data, tracking-number deduplication, Google Drive PDF generation, print notifications, and a history log.

Local Pickup Notification Flow

Local pickup orders needed a different customer experience than shipped orders. I connected the warehouse trigger to the customer notification flow so pickup-ready messages, reminders, cancellations, refunds, and pickup-complete messages happen consistently.

The important part was preventing the wrong message at the wrong time. Processed-state tags and fulfillment stop conditions keep reminders from repeating after an order is picked up, cancelled, or completed.

Built with ShipStation, Make.com, Shopify, and Shopify Flow, with clear tags for each notification state.
