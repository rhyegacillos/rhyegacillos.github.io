AI Automation Engineering

Recent private-client operations automation work focused on replacing manual copy-paste, spreadsheet checking, and follow-up tasks with simple operator tools. Each project starts with the same goal: make the business process easier to run, harder to break, and clear enough that non-engineers can operate it without needing the implementation details.

Warehouse Inventory Sync

Goal: Turn warehouse scanning and inventory updates into one traceable workflow instead of separate manual steps across warehouse, spreadsheet, store, and inventory tools.

What it solves: Staff can scan an order, record item completion, attach proof, and mark the work done while the automation prepares the right inventory update path. It reduces missed deductions, duplicate updates, and unclear handoffs between warehouse and back-office operations.

How it works: AppSheet, Google Sheets, Apps Script, Shopify Admin API, and ERP/inventory APIs are connected through clear status states, source rules, audit logs, review dashboards, retry-safe actions, and operator approval steps.

B2B Account Import Tool

Goal: Make a large business-customer migration safe to run from a spreadsheet instead of forcing operators to create each account manually in an admin UI.

What it solves: Manual account setup is slow and error-prone. The tool validates rows before import, generates IDs, previews setup data, creates or updates account records, links existing contacts, applies ordering rules, and writes clear row-level results back to the sheet.

How it works: Google Sheets and Apps Script drive the workflow, with Shopify GraphQL calls behind dry-run mode, rate limiting, retry/backoff, idempotent re-runs, whole-row status coloring, actionable error messages, and append-only run logs.

Procurement Reorder Dashboard

Goal: Replace manual sales exports, inventory exports, and spreadsheet formulas with a dashboard that answers what to reorder, how many units to buy, and when to place the order.

What it solves: Purchasing decisions become easier to review because sales, inventory, open purchase orders, supplier lead times, kit breakdowns, and forecast logic are pulled into one place instead of being checked across several files.

How it works: Apps Script pulls channel and ERP data, normalizes item mappings, computes reorder recommendations, groups orders by supplier/factory, highlights data issues, supports discontinued-item controls, and presents the result in an operator dashboard.

Shipping Manifest Automation

Goal: Create a daily pickup manifest that operators can print, have signed, and keep as proof that packages were handed to the carrier.

What it solves: When tracking scans are delayed or missing, the business needs a reliable record of which packages left the warehouse. The workflow combines shipping labels and inbound shipment records into one signed document instead of relying on manual lists.

How it works: Apps Script pulls shipments from ShipStation and marketplace shipping APIs, deduplicates tracking numbers, separates account groups, generates a PDF in Google Drive, posts a print notification, and keeps history, hold, restore, and remove controls in the sheet.

Local Pickup Notification Flow

Goal: Convert a manual pickup-ready and reminder process into a repeatable workflow that keeps customers informed without staff sending each message by hand.

What it solves: Pickup orders need different messages than shipped orders, and reminders must stop once an order is picked up, cancelled, or refunded. The automation prevents duplicate reminders, missed pickup notifications, and incorrect customer messages.

How it works: ShipStation acts as the warehouse trigger, Make.com moves pickup-ready state into Shopify, and Shopify Flow handles ready, reminder, cancellation, refund, and completion messages using explicit processed-state tags and fulfillment stop conditions.
